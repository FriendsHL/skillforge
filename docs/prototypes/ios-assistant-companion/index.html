SFResearch Agent刚刚

手机端适合“可操作结果”，但不应替代所有 Markdown

结论不是二选一：短答案继续用 Markdown；当结果需要筛选、保存进度或反复操作时，再升级为 Personal App。

为什么在手机上可行

  • 触控更自然：筛选、收藏和滑块比长表格更适合窄屏。
  • 结果可持续：保存状态后，下次打开能从上次位置继续。
  • 风险可控制：离线、自包含页面配合受控 Bridge，不直接开放任意系统能力。
推荐原则：先让答案可读，再让高价值结果可操作；不要把每条回复都包装成 App。

Parity Checklist

基线来自 iPhone 17 Pro / iOS 26.5 deterministic XCUITest 截图。

✓

Navigation

Current 固定为 Chat / Control / Agents / Settings。

✓

Visual tokens

系统大标题、暖灰背景、白色分组、橙色强调、浮动 Tab Bar。

✓

Current surfaces

Chat header/status/composer、Control schedules、Agents roster、Settings health/device。

✓

Launch & loading

冷启动先显示品牌恢复页；Personal Apps 首次请求使用同构骨架卡片，不出现纯白空窗。

✓

Personal Apps Library

Current 入口位于 Control → Workspace；长摘要最多两行，卡片保留来源、时间、离线/权限事实和完整操作。

✓

Agent-first Chat

Current 让 Agent 回复接近全宽、用户 Query 使用柔和蓝紧凑气泡，并把 Tool、文件与 Personal App 结果嵌进答案流。

✓

Query Color

柔和蓝 B 已进入真实 App；历史暖色与饱和蓝只保留为设计决策记录。

✓

Markdown

Current 覆盖三级标题、段落、两种列表、引用、代码语言栏、复制反馈和系统链接。
