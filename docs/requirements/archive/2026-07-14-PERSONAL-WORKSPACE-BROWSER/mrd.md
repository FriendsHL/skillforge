# PERSONAL-WORKSPACE-BROWSER MRD

## 用户诉求

用户希望 SkillForge 作为个人助理时，不只在 Chat 中返回文本，还能理解“我有哪些资料”和“本次任务生成了哪些文件”。授权空间最大应到 `/Users/youren/myspace`，而不是固定在其中的 `skillforge` 仓库。

## 核心问题

- 当前会话附件能表达本轮产物，但无法浏览已有资料。
- 直接把文件树塞进 Chat 会挤压主对话，也容易误解为全部已进入模型上下文。
- 直接暴露任意文件系统路径会产生路径穿越、符号链接逃逸、秘密文件泄露和巨量扫描风险。
- SkillForge 面向个人助理，不应照搬纯 coding agent 的“仓库即全部工作区”模型。

## 产品判断

Workspace 应是 Chat 的伴随资料面板：保留对话主入口，同时让用户确认来源、浏览已有文件和定位生成产物。浏览与模型读取必须分离，用户仅查看文件不会隐式增加 token 或改变 Agent 状态。

## 成功标准

- 用户在不离开 Chat 的情况下区分本次产物和长期文件。
- 授权边界清楚，可从 MySpace 根浏览多个项目。
- 默认只读且最小暴露；服务端独立执行所有路径和忽略规则校验。
- 后续可以在此基础上增加显式引用，而不用重做数据边界。
