# PERSONAL-WORKSPACE-BROWSER PRD

## 目标

在 Dashboard Chat 右侧栏提供个人工作空间浏览器，使用户能查看本次会话文件和授权目录内已有文件，同时保持 Chat-first 和只读安全边界。

## 用户工作流

### 查看本次会话产物

1. 用户打开一个 Chat session。
2. 在右侧选择 `Workspace`，默认进入 `This session`。
3. 查看上传文件和 Agent 生成文件；支持已有附件预览/下载能力。

### 浏览 MySpace

1. 用户切换到 `MySpace`。
2. 从授权根开始浏览目录，用面包屑返回任意父级。
3. 在当前目录内过滤名称。
4. 点击可预览文件；Markdown 结构化显示，普通文本等宽显示。
5. 点击返回，保持当前位置与过滤交互稳定。

## 功能需求

- FR-1：显示明确的根 label、当前位置和当前目录条目数。
- FR-2：目录优先、名称稳定排序；大目录按服务端上限截断并提示。
- FR-3：仅对允许且大小受控的文本提供内容预览。
- FR-4：加载、空目录、不可用、读取失败和重试都有独立状态。
- FR-5：切换 session 时，本次会话附件查询跟随 session；MySpace 浏览查询按认证用户隔离缓存。
- FR-6：过滤只作用于当前已加载目录，不触发全盘扫描。
- FR-7：键盘焦点、按钮标签和状态提示满足基本可访问性。

## 安全需求

- SR-1：客户端路径永远不能成为文件系统绝对路径的信任来源。
- SR-2：拒绝绝对路径、NUL、`.`/`..` 越界和符号链接逃逸。
- SR-3：不返回真实授权根绝对路径。
- SR-4：隐藏 `.git`、环境变量文件、凭据、私钥、构建输出和嵌套仓库 ignored 文件。
- SR-5：所有响应禁止浏览器缓存；错误不包含服务端异常或绝对路径。

## 非功能需求

- 目录列表上限默认 500；文本预览上限默认 256 KiB。
- 文件系统错误不能让 Chat 页面白屏。
- 不递归扫描目录；单次请求只处理当前层。

## 验收证据

- 后端单元/Controller 测试覆盖正常浏览、越界、symlink、ignore、敏感文件、截断和错误映射。
- 前端 API/组件测试覆盖双视图、导航、过滤、预览和错误恢复。
- Dashboard 类型检查、生产构建与真实浏览器交互通过。
